项目技术文档
1. 项目介绍
本项目是一个从零实现的 TCP/IP 协议栈工程，并提供了多个运行/移植目标：

code/pc：PC 端测试工程，基于 pcap/Npcap 收发以太网帧。
code/src：通用协议栈源码与基础测试应用。
code/stm32：将协议栈移植到 STM32F407 平台的工程。
code/x86os-with-net：将协议栈移植到 32 位 x86 OS，并支持 RTL8139 网卡。
code/chapter：课程章节版本源码，用于学习演进过程。
协议栈覆盖链路层、网络层、传输层和部分应用层工具，包含 ARP、IPv4、ICMPv4、RAW Socket、UDP、TCP、DNS、Ping、TFTP、HTTP、NTP、Echo 等实现。

2. 技术架构
项目整体属于“分层协议栈 + 平台适配层 + 应用测试层”架构。

主要分层：

应用层：ping、tftp、httpd、ntp、echo
Socket API 层：netapi.c、socket.c
Sock 抽象层：sock.c，将 RAW/UDP/TCP 统一成 sock_ops_t
传输层：raw.c、udp.c、tcp.c、tcp_in.c、tcp_out.c、tcp_state.c
网络层：ipv4.c、icmpv4.c、dns.c
链路层：ether.c、arp.c
网络接口层：netif.c
基础设施：pktbuf.c、mblock.c、fixq.c、nlist.c、timer.c
平台层：sys_plat.c、net_plat.c、netif_pcap.c、netif_disc.c
核心调度模型是单核心协议栈线程：外部网卡输入和应用 Socket API 请求都通过 exmsg 消息队列进入核心线程处理。

4. 核心模块说明
初始化模块：net.c 提供 net_init() 和 net_start()。
核心消息模块：exmsg.c 创建协议栈工作线程，处理网卡输入和 API 请求。
网络接口模块：netif.c 管理网卡打开、激活、输入队列、输出队列和默认路由。
数据包模块：pktbuf.c 使用链式块管理变长网络包。
内存块模块：mblock.c 用固定大小内存块管理协议控制块。
以太网模块：ether.c 处理 Ethernet 帧收发与 ARP/IP 分发。
ARP 模块：arp.c 管理 IP 到 MAC 的缓存、请求、响应和超时。
IPv4 模块：ipv4.c 处理路由表、IP 输入输出、分片与重组。
ICMPv4 模块：icmpv4.c 支持 Echo Reply 和不可达报文。
Socket API 模块：socket.c 实现 x_socket/x_send/x_recv/x_bind/x_connect/x_listen/x_accept。
UDP 模块：udp.c 实现 UDP 控制块、端口分配、收发队列。
TCP 模块：tcp.c/tcp_in.c/tcp_out.c/tcp_state.c/tcp_buf.c 实现连接管理、状态机、收发缓存、ACK、FIN、重传、Keepalive。
DNS 模块：dns.c 通过内部 UDP 控制块发送 DNS 查询并维护缓存。
应用模块：ping、tftp、httpd、ntp、echo 基于自定义 socket API 运行。
5. 系统流程
启动流程来自 code/pc/src/app/test/main.c：

调用 net_init() 初始化平台、工具、消息、包缓存、接口、以太网、定时器、ARP、IPv4、ICMP、Socket、RAW、UDP、DNS、TCP、Loopback。
调用 netdev_init() 打开 pcap 网卡，设置 IP、掩码、网关并激活。
调用 net_start() 启动 exmsg 核心线程。
启动 UDP Echo、TFTP Server、TCP Echo Server、HTTP Server。
进入命令循环，支持 ping、tftp-get、tftp-put、tftp、time、get。
6. 数据流
接收方向：

pcap/网卡中断
→ netif_put_in()
→ exmsg_netif_in()
→ exmsg 核心线程
→ netif_get_in()
→ ether_in()
→ arp_in() 或 ipv4_in()
→ icmpv4_in()/udp_in()/tcp_in()/raw_in()
→ sock 接收队列或 TCP 接收缓存
→ 应用 recv/recvfrom/read
发送方向：

应用 send/sendto/write
→ socket.c
→ exmsg_func_exec()
→ sock_*_req_in()
→ raw/udp/tcp 具体发送实现
→ ipv4_out()
→ netif_out()
→ ether_out()
→ arp_resolve() 或 ether_raw_out()
→ netif_put_out()
→ pcap/硬件发送线程
状态流：

Socket API 请求通过 NET_EXMSG_FUN 进入核心线程同步执行。
网卡输入通过 NET_EXMSG_NETIF_IN 进入核心线程异步处理。
TCP 状态流由 tcp_state.c 管理，包含 CLOSED/LISTEN/SYN_SENT/SYN_RECVD/ESTABLISHED/FIN_WAIT_1/FIN_WAIT_2/CLOSING/TIME_WAIT/CLOSE_WAIT/LAST_ACK。
TCP 输出状态由 tcp_out.c 管理，包含 IDLE/SENDING/REXMIT。
7. API 文档
公开初始化 API：

net_err_t net_init(void);
net_err_t net_start(void);
Socket API：

int x_socket(int family, int type, int protocol);
ssize_t x_sendto(int sid, const void* buf, size_t len, int flags, const struct x_sockaddr* dest, x_socklen_t dest_len);
ssize_t x_recvfrom(int sid, void* buf, size_t len, int flags, struct x_sockaddr* src, x_socklen_t* src_len);
int x_connect(int sid, const struct x_sockaddr* addr, x_socklen_t len);
int x_bind(int sid, const struct x_sockaddr* addr, x_socklen_t len);
ssize_t x_send(int fd, const void* buf, size_t len, int flags);
ssize_t x_recv(int fd, void* buf, size_t len, int flags);
int x_listen(int sockfd, int backlog);
int x_accept(int sockfd, struct x_sockaddr* addr, x_socklen_t* len);
int x_close(int sockfd);
netapi.h 会将标准名字映射到自定义实现，例如 socket -> x_socket、send -> x_send、recv -> x_recv、connect -> x_connect。

int tftp_start(const char *ip, uint16_t port);
int tftp_get(const char *ip, uint16_t port, int block_size, const char *filename);
int tftp_put(const char *ip, uint16_t port, int block_size, const char *filename);
int httpd_start(const char *dir, uint16_t port);
int tcp_echo_server_start(int port);
int tcp_echo_client_start(const char *ip, int port);
net_err_t udp_echo_server_start(int port);
int udp_echo_client_start(const char *ip, int port);
struct tm *ntp_time(void);

11. 部署方式
PC 端：

安装 Npcap/pcap 相关库。
使用 code/pc/CMakeLists.txt 构建。
Windows 下链接 wpcap、packet、Ws2_32。
启动生成的 net.exe。
确认 sys_plat.h 中 netdev0_phy_ip 对应本机真实网卡 IP。
STM32 端：

使用 code/stm32/USER/tcpip.uvprojx 这类 Keil 工程文件。
平台依赖包含 STM32 标准外设库、LAN8720、以太网 DMA、项目内 TinyOS 风格 OS 抽象。
netif_disc.c 负责 STM32 以太网驱动接入。
x86 OS 端：

code/x86os-with-net/README.md 说明该目录实现了可在 QEMU 下运行的 32 位 x86 OS，并移植自研 TCP/IP 协议栈。
使用 RTL8139 网卡。
应用可通过 socket 相关系统调用开发网络程序。

13. 常见问题
PCAP 找不到网卡：检查 netdev0_phy_ip 是否是本机真实网卡 IP。
Windows 运行缺少 Npcap：需要安装 Npcap，并确保 DLL 路径可被加载。
UDP 收不到包：udp_find() 依赖本地端口、本地 IP、远端 IP、远端端口匹配。
TCP 发送不一定立即到达：x_send() 写入 TCP 发送缓存后，由 TCP 输出状态机负责发送和重传。
DNS 解析失败：DNS 请求固定向 8.8.8.8 发送，依赖路由和外网连通性。
TFTP 传输异常：TFTP 使用 UDP 和超时重传，块大小与选项协商不一致会失败。
HTTP 只支持 GET：httpd.c 对非 GET 返回 501。
