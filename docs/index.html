Layered Architecture

Core protocols, platform adapters, and test applications are split into clear layers. This improves debug efficiency on PC and lowers migration friction to embedded and OS projects.

Platform Map

The repository keeps separate project paths for PC verification, STM32F407 migration, and x86 OS integration. This is one of the strongest engineering assets of this codebase.

Packet Processing Flow

Incoming frames and socket-side requests both converge into the core processing path, making behavior easier to trace during protocol debugging.

Quick Start

For first-run validation, start from `code/pc`. Keep working directory at `code/pc/work` so HTTP static file loading uses the expected `htdocs` path.

Recommended path

  • Build with `scripts/build-pc.ps1`
  • Run with `scripts/run-pc.ps1`
  • Adjust host network values in `code/pc/src/plat/app_cfg.h`
# Windows PowerShell
./scripts/build-pc.ps1
./scripts/run-pc.ps1

# or manual
cd code/pc
cmake -S . -B build
cmake --build build
cd work
../build/net
