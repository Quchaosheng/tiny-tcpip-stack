# Release v0.2.1 (P0/P1 Security and Robustness Hardening)

Release Date: 2026-05-25

This release focuses on closing high-priority risks and hardening input boundaries without changing core business behavior. It is intended as a stabilization update after `v0.2.0`.

## Highlights

- Fixed incorrect HTTP version validation logic. The server now correctly accepts `HTTP/1.0` and `HTTP/1.1`.
- Fixed HTTP request buffer termination to use the actual `recv` length before appending `\0`.
- Replaced key `sprintf` usages in HTTP response/path assembly with `snprintf`, plus truncation checks.
- Added URL safety checks to block obvious path escape patterns.
- Replaced unbounded `scanf("%s %s")` with `fgets + width-limited sscanf` to prevent command input overflow.
- Hardened TFTP request parsing with bounded string parsing and null-termination checks.
- Added TFTP filename validation to reject path traversal and unsafe separators.
- Hardened platform-layer string handling by updating `pcap_find_device` to use bounded copy with buffer-size input.
- Replaced BPF filter `sprintf` with `snprintf` and added result validation.

## Scope

- HTTP server module: `code/pc/src/app/http/httpd.c`
- TFTP server module: `code/pc/src/app/tftp/tftp_server.c`
- Test app command parser: `code/pc/src/app/test/main.c`
- Platform adapter: `code/pc/src/plat/sys_plat.c`, `code/pc/src/plat/sys_plat.h`

## Compatibility Notes

- No new external dependencies were introduced.
- External behavior remains compatible; invalid input handling is now stricter.
- Function signature update: `int pcap_find_device(const char* ip, char* name_buf, size_t name_buf_size);`

## Known Limitations

- Full local build validation was not completed in the current environment due to missing/unaligned toolchain and CMake generator setup.
- Recommended: run a full build and integration check in a properly configured Windows build environment before tagging the final release.
